# Prompt: HackSuite Landing Page

## Role
You are an experienced web designer and front-end developer. Apply modern UI/UX best practices, clean typography, intentional whitespace, and a strong visual hierarchy throughout.

---

## Objective
Build a polished, standalone landing page for **HackSuite** — a purpose-built SaaS platform for university hackathon organizers. This page serves as the public-facing entry point that introduces the product and directs visitors to the existing platform features already developed.

---

## Design Spec

**Color Palette:**
- Primary: Deep rose / dusty pink (`#C2185B` or similar)
- Secondary: Charcoal black (`#1A1A1A`)
- Neutral: Warm gray (`#6B6B6B`, `#E8E4E1`)
- Accent: Dark maroon (`#7B1C3E`) for hover states and highlights

**Aesthetic:** Calm, classy, and modern. Think editorial luxury — not flashy. Clean lines, minimal decoration, strong contrast between dark backgrounds and light text.

**Typography:**
- Headings: A serif or elegant sans-serif (e.g., Playfair Display, DM Serif, or similar)
- Body: Clean sans-serif (e.g., Inter, DM Sans)

**Layout:** Responsive, mobile-first. Sections should stack gracefully on small screens.

---

## Page Structure

### 1. Hero Section
- Full-width, dark background (charcoal or near-black)
- HackSuite logo or wordmark
- One-line tagline (derived from mission statement)
- Single CTA button → links to the existing platform dashboard/app

### 2. Mission Statement Section
- Short, centered block of text
- State clearly what HackSuite is, who it's for, and why it exists
- Subtle pink/maroon accent divider above or below

### 3. Feature Highlights
- 3–4 cards or icon blocks summarizing the core modules already built
- Use icons + short labels (Registration, Check-in, Judging, Sponsors, etc.)
- Cards on dark gray background with pink accent on hover

### 4. Navigation / Links to Existing Pages
- A grid or list of links to already-developed sections of the platform
- Each link styled as a pill or card with a short description
- Make it obvious these are live, working features

### 5. Footer
- Simple: logo, tagline, copyright
- Optional: GitHub link, contact email

---

## Technical Requirements
- Pure HTML/CSS (or React if preferred) — no heavy frameworks unless justified
- No placeholder Lorem Ipsum — write real copy based on the HackSuite mission
- All navigation links should be relative paths to existing pages in this project
- Smooth scroll behavior, subtle fade-in animations on scroll (CSS only preferred)
- Fully accessible: semantic HTML, proper heading hierarchy, alt text, ARIA labels where needed

---

## Deliverable
A single `index.html` (or `landing.tsx` / `landing.jsx` if React) with scoped styles. The page must look production-ready — not a wireframe or prototype.
